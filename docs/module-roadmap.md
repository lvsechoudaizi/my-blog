# 模块路线图草稿

## 文档目的
这份文档用于说明当前 `my-blog` 项目下一阶段适合继续扩展的模块方向，避免后续开发只凭即时想法推进，导致模块边界越来越模糊。

## 当前已完成基础

### 前端
- 路由基础结构
- 登录页
- 首页占位页
- 404 页面
- Pinia 登录态管理
- Less 样式目录化
- 登录流程与用户信息获取

### 后端
- `common` 公共模块
- `auth-center` 认证服务基础能力（登录、当前用户信息）
- `blog-service` 示例业务接口
- `gateway` 路由与 JWT 鉴权
- JWT令牌验证与用户信息透传

### 基础设施
- MySQL
- Redis
- Nacos
- Docker Compose 本地运行方案

## 路线图阶段划分

### 第一阶段：把现有链路做稳
目标：
- 保证本地全链路稳定启动
- 保证登录、鉴权、透传链路稳定
- 保证前端基础结构和样式体系稳定

建议继续补的内容：
- 修复和固化 Nacos 连接问题
- 增加当前用户信息接口
- 增加主题切换能力
- 增加首页真实数据占位

### 第二阶段：补齐后台基础能力
目标：
- 从“脚手架”进入“可用后台”

建议模块：
- 菜单管理
- 角色权限管理
- 用户管理
- 面包屑和导航体系
- 基础仪表盘

### 第三阶段：进入博客业务能力
目标：
- 让项目开始承载真实博客后台管理场景

建议模块：
- 文章列表
- 文章新增和编辑
- 分类与标签管理
- 草稿箱
- 发布状态管理

### 第四阶段：进入项目展示与内容配置
目标：
- 支持个人官网和项目展示内容维护

建议模块：
- 项目管理
- 项目标签和技术栈管理
- 首页配置
- 关于页内容配置
- 站点基础信息配置

## 前端路线建议
- 增加业务组件分层
- 增加主题切换 store
- 增加统一消息提示能力
- 增加基础表格、表单和弹窗规范
- 增加页面级接口模型和空状态规范

## 后端路线建议
- 增加真实用户和角色模型
- 增加权限点控制
- 增加刷新 token 或会话续期策略
- 增加统一错误码
- 增加配置中心正式使用方式

## 网关路线建议
- 增加更清晰的公开接口维护机制
- 增加鉴权失败统一响应结构
- 增加更细粒度的用户上下文透传
- 增加请求日志和链路追踪能力

## 文档路线建议
- 每增加一个核心模块，同步补充接口约定
- 每解决一个真实坑，同步补充排障文档
- 每形成一个可复用流程，同步补充 `SKILL.md`

## 当前优先级建议
如果按投入产出比排序，当前最值得优先推进的是：
1. 稳定本地启动和联调
2. 搭建后台基础导航和仪表盘
3. 开始博客文章管理
4. 增加主题切换能力

## 暂时不建议过早投入的内容
- 过度复杂的权限系统
- 过早拆分过多微服务
- 过度设计前端组件体系
- 过早引入过多基础设施

## 这份路线图的使用方式
- 新增需求前先看是否属于某个阶段
- 开发完成后回写到文档
- 如果优先级变化，直接调整这里的阶段内容

这份路线图应作为项目演进方向的草稿，不是一次性定死的计划。
